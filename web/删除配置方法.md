# 删除 ENavigate 配置的方法

本文档说明如何撤销 `https://本机域名/enavigate/index` 的 nginx 配置。

---

## 配置位置

nginx 配置位于：

```
/etc/nginx/sites-available/lobster
```

该文件通过符号链接被 `sites-enabled` 引用：

```
/etc/nginx/sites-enabled/lobster -> /etc/nginx/sites-available/lobster
```

---

## 删除步骤

### 1. 编辑 nginx 配置文件

```bash
sudo nano /etc/nginx/sites-available/lobster
```

### 2. 删除以下配置块

找到并删除这段内容：

```nginx
    # ENavigate 静态页面
    location /enavigate/ {
        alias /var/www/enavigate/;
        index index.html;
    }
    # 允许 /enavigate 和 /enavigate/index 访问到 index.html
    location = /enavigate {
        return 301 /enavigate/;
    }
    location = /enavigate/index {
        rewrite ^ /enavigate/ last;
    }
```

### 3. 测试并重载 nginx

```bash
sudo nginx -t          # 检查语法
sudo nginx -s reload   # 重载配置
```

### 4. 验证删除

访问 `https://myxiaxiais.art/enavigate/index`，应返回 **404 Not Found**。

---

## 一键删除脚本

```bash
# 备份原配置
sudo cp /etc/nginx/sites-available/lobster /etc/nginx/sites-available/lobster.bak.$(date +%Y%m%d_%H%M%S)

# 删除 ENavigate 相关配置块
sudo sed -i '/# ENavigate 静态页面/,/\s*}/d' /etc/nginx/sites-available/lobster
sudo sed -i '/location = \/enavigate/,/\s*}/d' /etc/nginx/sites-available/lobster

# 测试并重载
sudo nginx -t && sudo nginx -s reload
```

> ⚠️ 注意：一键脚本可能不完美，如果执行后出现问题，建议手动编辑。备份文件可随时恢复。

---

## 删除数据目录

删除 nginx 配置后，如果不需要再保留文件，可清理复制过来的数据：

```bash
sudo rm -rf /var/www/enavigate
```

---

## 恢复方法

如果误删，从备份恢复：

```bash
sudo cp /etc/nginx/sites-available/lobster.bak.xxx /etc/nginx/sites-available/lobster
sudo nginx -t && sudo nginx -s reload
```
